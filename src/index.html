Uhm, what's that?

Good you're asking.
To get straight to the point, this dApp verifies that your Ethereum Account is acctually an EOA (or Externally Owned Account) and NOT a smart contract. Let that sink in for a moment.

Why build this tool?

Consider the following question: Does an address belong to some smart contract or to some EOA? We can tell them apart fairly easy (contacting/knowing the owner may be enough). For other smart contracts, though, this is a tricky question to answer. Solving said problem results in a more-complex-than-necessary user experience, as well as unnecessary burdens for developers.

What is this useful for?

Soulbound Tokens for example.
Suppose a token with a lock-up period after buying it. Next, let's assume that, instead of some user, a smart contract handles the buying process. Finally, we add some functionality to pass on the ownership of said smart contract to someone else. All of a sudden, our seemingly locked-up token becomes transferable again.

Non transferable assets are only possible when smart contracts are banned from owning them. Imagine one could buy a degree in some subject without first studying it. Sounds stupid, but is, at least without a system like Outsider, unavoidable on Ethereum.

How does Outsider help?

To verify that some address is in fact an EOA, developers need to recover a public key from a so called ECDSA signature. While not as complicated as it sounds, such a procedure is not exactly trivial to implement.
Outsider hides this logic and thereby reduces the difficulty to a singular, copy-and-pastable, line of code.
This means multiple things:

A) (New) Users are a little bit less overwhelmed by all this arguably cryptic software.

B) The end product is more secure.

C) Developers can spend more time on actually building innovative software.
